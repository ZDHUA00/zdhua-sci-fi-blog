这次重建的边界很明确：Git 历史、GitHub Pages 工作流和 `zdhua.me` 的 CNAME 继续保留，博客实现本身清空重做。这样可以避免域名和部署链路被破坏，同时让界面从旧版本里脱离出来。

新的实现仍然使用 Astro。首页先呈现星舰控制台，再把文章作为航行日志放进档案舱。BGM 不再占住页面空间，而是放到顶部隐藏下拉里。

## 为什么清空旧实现

旧版本已经能部署，但视觉和性能被多个方向反复叠加，继续补丁会让样式越来越难控制。重建的收益是代码结构更清晰，动画预算也能重新分配。

这次把重型图片和音频文件都移除，改成代码生成的轻量视觉和 Web Audio 合成声道。页面加载更直接，也方便后续调整强度。

## 新的重点

第一屏必须有明确的控制台气质，但不能牺牲阅读。星域、扫描线、轨迹和信号图只做背景与状态表达，文章正文保持足够清晰。

移动端优先避免横向溢出。布局不会依赖固定宽度，也不会把音乐面板固定成挡住内容的浮层。

## 部署策略

GitHub Actions 仍然负责构建并发布到 Pages。构建环境里使用 `SITE_URL=https://zdhua.me` 和 `BASE_PATH=/`，所以产物天然适配自定义域名。

只要 DNS 和 GitHub Pages 自定义域名设置保持有效，推送到 `main` 后，`zdhua.me` 就会进入新的版本。
